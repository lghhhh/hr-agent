# AI智能招聘面试管理系统 - 全栈开发需求框架
> 适用对象：AI Coding智能体；目标：输出可直接部署上线的完整Web应用，实现招聘面试数据自动化、流程化管理

---

## 一、项目概述
### 1.1 项目背景
当前招聘流程依赖人工手动记录面试信息、更新候选人状态，存在录入效率低、数据易遗漏、同步不及时的问题。本系统以「最低开发&部署成本」为原则，基于面试对话文本实现AI自动结构化总结，串联多轮面试审批流程，替代手工录入，实现招聘数据自动化沉淀与可视化管理。

### 1.2 核心目标
1.  面试对话文本一键AI解析，自动生成结构化面试报告，替代人工手写记录
2.  多轮面试流程线上化流转，支持每轮评价+通过/淘汰决策，状态自动同步
3.  候选人全生命周期数据统一存储，支持检索、筛选、追溯
4.  招聘数据自动统计可视化，无需人工汇总报表
5.  支持公网部署，通过浏览器即可访问使用，适配远程协作场景

### 1.3 设计原则
- **最小成本**：全栈采用开源免费技术栈，最低配置服务器即可运行，无商业授权成本
- **快速落地**：优先实现核心流程，MVP版本可快速上线验证
- **兼容现有生态**：预留企业微信、腾讯在线文档对接扩展能力
- **可配置化**：AI模型、面试轮次、岗位信息均可通过配置调整，无需改代码

## 三、核心功能模块设计
### 模块1：AI面试结构化解析模块
**核心能力**：输入原始面试对话文本，调用大模型自动生成结构化面试报告
1.  输入方式：文本框粘贴面试全量对话记录（预留腾讯会议纪要API对接入口）
2.  AI解析输出固定字段：
    - 候选人基础信息提取（姓名、应聘岗位、工作年限、期望薪资等）
    - 核心技能匹配度分析
    - 项目经验亮点与疑点
    - 面试核心问答摘要
    - 候选人优势与不足
    - 综合初评建议
3.  支持人工修改AI生成的总结内容，确认后存入数据库
4.  内置系统级Prompt，保证输出格式稳定、信息全面，支持在后台自定义调整Prompt

### 模块2：多轮面试流程流转模块
**核心能力**：标准化串联初面→复面→终面全流程，状态自动流转
1.  轮次配置：支持自定义设置面试轮次（默认初面、复面、终面3轮，可增删）
2.  单轮操作流程：
    - 选择候选人→录入/导入面试对话→AI生成总结→面试官填写主观评价→选择「通过/不通过」→提交
    - 提交后不可修改，保证面试记录可追溯
3.  流转规则：
    - 当前轮次「通过」：候选人状态自动更新为「待下一轮面试」，进入下一轮面试队列
    - 当前轮次「不通过」：候选人状态标记为「已淘汰」，流程终止，记录淘汰原因
    - 终面「通过」：候选人状态标记为「拟录用」
4.  轮次数据隔离：每轮面试记录独立存储，后续轮次面试官可查看全部历史轮次的AI总结与面试官评价

### 模块3：候选人全生命周期管理模块
1.  候选人信息档案：基础信息、应聘岗位、简历附件/链接、全部面试轮次记录、当前状态、跟进时间
2.  列表筛选：支持按岗位、面试状态、面试轮次、时间范围、关键词检索
3.  状态标签：待初面、初面通过待复面、复面通过待终面、已淘汰、拟录用
4.  操作权限：支持新增、编辑、删除候选人信息

### 模块4：数据可视化统计模块
1.  招聘总览看板：累计候选人数量、各岗位招聘人数、整体通过率、各轮次淘汰率
2.  趋势统计：按日/周/月新增候选人数量、面试完成量趋势图
3.  岗位维度统计：各岗位的投递量、面试量、通过率、招聘进度
4.  支持统计数据导出为Excel

### 模块5：系统账号与权限模块
- MVP版本：默认管理员账号，全功能访问权限
- V1.0版本：支持多账号管理，分为「管理员」「面试官」两种角色
  - 管理员：全功能权限，可管理账号、岗位、查看全部数据
  - 面试官：仅可查看自己负责的面试、提交面试评价，不可删除数据

### 模块6：导出与外部协同模块
1.  单候选人面试报告导出：支持导出为PDF/Excel，包含全部轮次的AI总结与面试官评价
2.  全量候选人名单导出：按筛选条件导出为Excel表格
3.  预留接口能力：支持后续对接企业微信机器人、腾讯在线文档API，实现数据自动同步

## 五、后端API接口规范
> 统一返回格式：`{code: 200, msg: "success", data: {}}`
> 全部接口需支持跨域，携带Token鉴权

### 5.1 认证接口
1.  `POST /api/auth/login`：账号密码登录，返回Token
2.  `GET /api/auth/userinfo`：获取当前登录用户信息

### 5.2 候选人管理接口
1.  `GET /api/candidate/list`：分页获取候选人列表，支持筛选
2.  `POST /api/candidate/add`：新增候选人
3.  `PUT /api/candidate/update`：编辑候选人基础信息
4.  `DELETE /api/candidate/delete`：删除候选人
5.  `GET /api/candidate/detail/{id}`：获取候选人详情+全部面试轮次记录

### 5.3 面试流程接口
1.  `POST /api/interview/create`：创建新一轮面试记录
2.  `PUT /api/interview/update-summary`：更新AI生成的面试总结（人工修改后提交）
3.  `POST /api/interview/submit`：提交面试结果（评价+通过/不通过），触发状态流转
4.  `GET /api/interview/detail/{id}`：获取单轮面试详情

### 5.4 AI解析接口
1.  `POST /api/ai/generate-summary`：传入面试对话文本+岗位信息，调用大模型生成结构化总结
    - 请求参数：`dialogue_text`、`position_id`、`candidate_name`
    - 返回：结构化JSON格式的面试总结

### 5.5 统计看板接口
1.  `GET /api/stats/overview`：获取首页看板总览数据
2.  `GET /api/stats/trend`：获取面试量趋势数据
3.  `GET /api/stats/position`：获取各岗位招聘统计数据

### 5.6 导出接口
1.  `GET /api/export/candidate-list`：导出候选人列表Excel
2.  `GET /api/export/interview-report/{candidate_id}`：导出单人面试报告PDF

## 七、部署上线要求
### 7.1 服务器最低配置
- 1核2G内存 Linux轻量应用服务器（CentOS/Ubuntu均可）
- 带宽1M以上，支持公网IP
- 成本：每月几十元，满足初期使用

### 7.2 部署交付要求
1.  提供 `Dockerfile` + `docker-compose.yml`，执行 `docker-compose up -d` 即可一键启动全服务
2.  前端打包后由FastAPI挂载静态文件，单容器同时提供前后端服务
3.  数据库文件挂载到宿主机，支持一键备份
4.  提供环境变量配置文件 `.env`，大模型密钥、端口、账号密码均可通过配置文件修改
5.  支持配置域名反向代理，适配HTTPS访问

### 7.3 访问方式
- 部署完成后，通过 `http://服务器IP:端口` 或绑定的域名直接在浏览器访问，无需安装客户端

## 九、开发约束
1.  代码结构清晰，前后端分离目录，附带完整README说明文档
2.  所有敏感配置（数据库、API密钥）必须通过环境变量读取，禁止硬编码
